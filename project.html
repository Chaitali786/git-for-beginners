Project Page - Git and GitHub for Beginners

Why Remote Repositories Are Important

Remote repositories are necessary for collaboration, serving as a central hub for teams to share and synchronize code. They provide a critical backup of the project, protecting against local data loss and ensuring progress isn't tied to a single machine

Steps to Create a Project on GitHub

  1. Go to GitHub.com.
  2. Log in to your account or create a free one.
  3. Click the + button in the top right and choose New repository.
  4. Enter a Repository name (for example: my-project).
  5. Write a short description (optional).
  6. Choose if your repo will be Public (everyone can see it) or Private (only you can see it).
  7. Click Create repository.
  8. Copy the web address (URL) that looks like this: https://github.com/yourusername/my-project.git.

Git Commands are need

1. Create a Project in Computer

Use these commands in terminal or Git Bash:

# Create a new folder for the project
mkdir my-project
cd my-project

# Start a new Git repository
git init

# Create a file called README.md
echo "# My Project" > README.md

# Save your changes
git add README.md
git commit -m "First commit"

2. Link the Local Project to GitHub

# Connect the  local project to GitHub
git remote add origin https://github.com/yourusername/my-project.git

# Rename branch to main (if needed)
git branch -M main

# Upload your code to GitHub
git push -u origin main

How Other People Can Help with Your Project

1. If You Add Them as a Collaborator

Collaborators can make changes directly to your GitHub project.

  1. Go to your GitHub repo → Settings → Collaborators.
  2. Add your friend’s GitHub username or email.
  3. They can copy your project to their computer using:
git clone https://github.com/yourusername/my-project.git
  1. They can make changes, save them, and send them back to GitHub with git push.

2. If Your Project Is Public (Open to Everyone)

If your project is public, anyone can help you improve it. Here’s how it works in simple steps:

  1. Someone clicks the Fork button on your GitHub page to make their own copy of your project.
  2. They make changes or add new features in their copy.
  3. When they finish, they click Pull Request to ask you to add their changes to your project.
  4. You look at their changes and decide whether to accept them or not.

This is how people work together on open-source projects on GitHub.

Tip

Always include a file called README.md. It explains what your project does and how others can use it.
