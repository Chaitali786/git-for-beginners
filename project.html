Git and GitHub for Beginners

Why Remote Repositories Are Important

Remote repositories are necessary for collaboration, serving as a central hub for teams to share and synchronize code...

Steps to Create a Project on GitHub

  1. Go to GitHub.com and log in.
  2. Click on New Repository.
  3. Enter a name and optional description.
  4. Choose Public or Private.
  5. Click Create repository.
Tip: Always include a file called README.md.

Git Commands You Need

1. Create a Project on Your Computer


# Create a new folder for the project
mkdir my-project
cd my-project

# Start a new Git repository
git init

# Create a file called README.md
echo "# My Project" > README.md

# Save your changes
git add README.md
git commit -m "First commit"

2. Link the Local Project to GitHub


# Connect the local project to GitHub
git remote add origin https://github.com/yourusername/my-project.git

# Rename branch to main (if needed)
git branch -M main

# Upload your code to GitHub
git push -u origin main

How Other People Can Help with Your Project

1. If You Add Them as a Collaborator

  • Go to your GitHub repo → Settings → Collaborators.
  • Add your friend’s GitHub username or email.
  • They can copy your project using: git clone https://github.com/yourusername/my-project.git
  • They can make changes, save them, and send them back to GitHub with git push.

2. If Your Project Is Public

  • Anyone can click the Fork button to make their own copy of your project.
  • They make changes or add new features in their copy.
  • When finished, they click Pull Request to ask you to add their changes.
  • You review and decide whether to accept their changes.
Tip: Always include a file called README.md. It explains what your project does and how others can use it.
